Privacy Policy

Last updated: 2026-05-16

Short version: AI Memory Reader and the /aimr Claude Code plugin run entirely on your device. They do not collect, transmit, or share any personal data, file content, or usage telemetry. There is no server, no analytics, no tracking.

1. Scope

This policy covers two products published by the AI Memory Reader project:

  • AI Memory Reader — the native macOS & iOS app distributed via GitHub Releases.
  • AI Memory Reader plugin for Claude Code — the /aimr slash command distributed via github.com/nvwalj/aimr-claude-plugin and the Anthropic plugin marketplace.

2. What data the products handle

Both products are read/edit tools for markdown files that already live on your local device or filesystem (such as CLAUDE.md, MEMORY.md, or files in ~/.claude/, ~/.openclaw/, etc.). File content is read directly from disk and rendered locally.

3. What data the products send over the network

None.

  • The macOS/iOS app has no analytics, no crash reporting service, no telemetry endpoints, and no remote configuration.
  • The /aimr Claude Code plugin is a single slash command that runs the local open shell command to dispatch a aimemoryreader:// URL on your machine. It does not make HTTP requests and does not transmit the file path or heading argument to any third party.
  • The only network activity the app may perform is fetching images explicitly embedded in markdown you open (e.g. ![](https://example.com/foo.png)). This is the same behavior as any markdown renderer and is initiated by content you choose to view.

4. What data we collect about you

None. We do not run a backend service. We do not have user accounts. We do not see, store, or process any information about your device, files, prompts, AI agent memory, or usage patterns.

5. Third-party services

  • GitHub hosts the source code, releases, and this website. GitHub's own privacy policy applies when you download a release or view this page: GitHub privacy statement.
  • Ko-fi processes donations on ko-fi.com/nvwalj. Donations are entirely optional and handled by Ko-fi and its payment processors. We do not receive payment-card data. See Ko-fi's privacy policy.

The app itself does not contact Ko-fi or GitHub. Visiting those services is initiated only when you click the corresponding link.

6. Permissions the app requests

  • File access — required to open and render the markdown files you choose. Files are read on demand; nothing is uploaded.
  • URL scheme handler registration — so that aimemoryreader:// links (used by the /aimr plugin and other agents) can open the app. Handling a URL scheme does not transmit data anywhere.

7. Children

These products are general-purpose developer utilities and are not directed to children under 13. We do not knowingly collect data from anyone.

8. Changes

If this policy ever changes, the updated text will appear at this URL and the "Last updated" date above will change. Material changes will also be noted in the GitHub release notes.

9. Contact

Questions or concerns: open an issue at github.com/nvwalj/ai-memory-reader/issues.
